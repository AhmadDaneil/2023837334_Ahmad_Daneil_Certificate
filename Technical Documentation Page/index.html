Introduction

This section introduces C++, a high-performance, general-purpose programming language.

C++ is widely used for system/software development, game engines, and real-time simulations.

#include <iostream>
int main() { std::cout << "Hello, C++!" << std::endl; return 0; }
  • Compiled Language
  • Supports OOP and Generic Programming
  • Strong performance and control over system resources
Getting Started

To get started with C++, install a compiler like GCC or use an IDE such as Visual Studio or Code::Blocks.

// Compile with g++
g++ main.cpp -o main

  • Install a C++ compiler
  • Set up an IDE or use command line
  • Write and compile your first program
Syntax

C++ syntax is derived from C with additions for object-oriented and generic programming.

int add(int a, int b) { return a + b; }

  • Data types and variables
  • Control structures (if, switch, loops)
  • Functions and prototypes
Pointers

Pointers store memory addresses and are essential for dynamic memory and performance-critical code.

int x = 10;
int* p = &x;
std::cout << *p;
  • Pointer arithmetic
  • NULL and nullptr
  • Pointer to pointer
Classes and Objects

Classes encapsulate data and behavior. C++ supports full object-oriented programming.

class Car {
  public:
    string brand;
    void drive() { std::cout << "Driving"; }
};
Car c;
c.drive();
  • Constructors and destructors
  • Inheritance and polymorphism
  • Access modifiers
Memory Management

C++ allows manual memory control using new and delete, which requires careful handling.

int* arr = new int[10];
// use arr
delete[] arr;
  • Heap vs Stack
  • Smart pointers (unique_ptr, shared_ptr)
  • Memory leaks and tools to detect them
Templates

Templates allow functions and classes to operate with generic types.

template <typename T>
T max(T a, T b) { return (a > b) ? a : b; }
std::cout << max(4, 7);
  • Function templates
  • Class templates
  • Template specialization
Best Practices

Following best practices improves code readability and maintainability.

// Always initialize variables
  • Use RAII (Resource Acquisition Is Initialization)
  • Avoid raw pointers when possible
  • Use const and references appropriately
